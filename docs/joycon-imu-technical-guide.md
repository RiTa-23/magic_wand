## 1. 全体の考え方

Joy-Conから取得できるIMUデータは**角速度（Gyro）**と**加速度（Accel）**です。
本プロジェクトにおける「魔法の杖（Wand）」のトラッキングでは、主に**ジャイロセンサー**を使用してJoy-Conの傾き（回転）の変化を蓄積し、画面上の2Dカーソル位置へと変換します。

### Joy-Conの持ち方（基準姿勢）

ユーザーにはJoy-Con (R) の **Rボタンがある側面を前方（杖の先）** として持たせます。
この向きに合わせた軸の再マッピングが必要です。

## 2. キャリブレーション（ジャイロバイアスの補正）

IMUセンサー、特にジャイロセンサーは個体差や温度によって静止時でも0ではない値（ドリフト／バイアス）を出力します。これを相殺するためのキャリブレーション処理を実装しています。

### キャリブレーションのフロー

1. **静止判定**:
   加速度センサーの値(`accel.x, accel.y, accel.z`)の前フレームとの差分を監視します。差分が閾値（`CALIBRATION_ACCEL_THRESHOLD = 500`）を超えた場合、デバイスが「動かされた」と判定し、キャリブレーションのタイマーをリセットします。
2. **データの収集**:
   静止状態が維持されている間、毎フレームのジャイロの値を配列に蓄積します。
3. **バイアスの算出**:
   3秒間（`CALIBRATION_DURATION = 3000` ms）静止が続いた場合、収集したジャイロ値のX, Y, Zそれぞれの平均値を計算し、`gyroBias` として保存します。
4. **状態の確定**:
   キャリブレーション状態を `done` に遷移させ、以降のトラッキングを開始します。

※ キャリブレーション中は、ユーザーに対して「SL/SRボタン側を下にして平らな場所に静止させてください」とUI上で誘導します。

## 3. センサーデータのマッピングとデッドゾーン

キャリブレーション完了後のトラッキングフェーズでは、取得した生データからバイアスを差し引き、カーソル移動量へと変換します。

### 軸のマッピング

「Rボタンを前」に向けた持ち方に対応するため、以下のように軸を割り当てます。

- **左右の動き (Yaw)**: `gyro.y` （Y軸の回転）
- **上下の動き (Pitch)**: `gyro.z` （Z軸の回転）

### デッドゾーンと感度調整

微細な手の震えや、取り切れなかった微量のノイズによるカーソルの「プルプル」を防ぐため、デッドゾーン（不感帯）を設けています。

```typescript
// バイアスを引く
let yaw = joyconState.imu.gyro.y - gyroBias.y;
let pitch = joyconState.imu.gyro.z - gyroBias.z;

// デッドゾーン処理 (GYRO_DEADZONE = 10)
if (Math.abs(yaw) <= GYRO_DEADZONE) yaw = 0;
if (Math.abs(pitch) <= GYRO_DEADZONE) pitch = 0;

// カーソル座標の更新 (GYRO_SENSITIVITY = 0.15)
pos.x -= yaw * GYRO_SENSITIVITY;
pos.y -= pitch * GYRO_SENSITIVITY;
```

## 4. トラッキング中のリセット機能

ジャイロセンサーは時間経過とともに積分誤差（ドリフト）が蓄積し、カーソルが意図しない方向へズレていくことがあります。
これを補正するため、ユーザー操作による即時リセット機能を設けています。

**プラス（＋）ボタンによるリセット:**
トラッキング中にJoy-Conの `+` ボタンを押下すると、以下の処理が実行されます。

1. 現在のカーソル位置・追跡軌跡をリセット（中心へ）
2. スムージング用のカメラ座標(`viewBoundsRef`)をクリア
3. 現在のジャイロバイアスを破棄し、自動的に再度「3秒間のキャリブレーション（静止待機）」状態へと戻る

これにより、ユーザーは画面上のボタンをクリックしなくても、手元の操作だけで即座にトラッキングのズレを直し、再スタートさせることができます。

## 5. 描画とスムージング処理 (LERP)

IMUの座標は空間的制限を持たないため、ユーザーが杖を振り続けるとXY座標は無限に大きくなります。
画面内に軌跡（Trail）を適切に収めるために、Canvas描画時に**動的スケーリングとスムージング**を行っています。

### 動的スケーリングとアスペクト比の維持

現在の軌跡の最小値・最大値(`minX, maxX, minY, maxY`)から描画すべき範囲を計算します。
この際、UI上で描いた図形（円など）が歪んで見えないよう、Canvasのサイズ（幅・高さ）のアスペクト比に合わせて、短い辺の表示領域を同等の比率になるように拡張します。（スケールだけを動的に変化させ、アスペクト比は固定）

### LERP（線形補間）によるカメラ追従

軌跡が更新されるたびにカメラの表示範囲が即座に切り替わると、画面がカクカクして非常にUXが悪化します。
これを解決するため、前フレームの表示範囲から目標となる表示範囲へ毎フレーム一定割合（例: 15%）だけ近づける処理（LERP）を行っています。

```typescript
const LERP = 0.15;
viewBounds.minX += (target.minX - viewBounds.minX) * LERP;
// (他軸も同様)
```

これにより、カーソルが画面外へ広がろうとする際のUIの「ズームアウト」や、古い軌跡が消えた際の「ズームイン」が、非常に滑らかなアニメーションとして描画されます。
